# Oracle Database Profil Yapılandırması ve Şifre Politikaları

Oracle veritabanında profiller, kullanıcıların kaynak kullanımını sınırlandırmak ve şifre politikalarını (password complexity, expiration, account lockout vb.) yönetmek için kullanılır. Güvenlik denetimleri (security audit) sırasında en çok incelenen alanlardan biri kullanıcı profilleridir.


<!--truncate-->
## Profil Nedir?

Bir profil, belirli bir veritabanı kullanıcısına atanabilen bir dizi kısıtlamadır. İki ana kategoriye ayrılır:
1. **Kaynak Limitleri (Resource Limits):** CPU zamanı, oturum süresi, eşzamanlı oturum sayısı vb.
2. **Şifre Parametreleri (Password Parameters):** Şifre geçerlilik süresi, hatalı giriş deneme sayısı, şifre zorluğu vb.

## Tüm Profillerin Listelenmesi

Veritabanındaki tüm profilleri ve bu profillerin barındırdığı kuralları `DBA_PROFILES` view'i üzerinden görebiliriz.

### SQL Sorgusu: Profil Detayları

```sql
set line 150 
set pagesize 1000

SELECT profile,
       resource_name,
       resource_type,
       limit
FROM dba_profiles
ORDER BY profile, resource_type;
```

## Kritik Şifre Politikası Parametreleri

Güvenlik açısından aşağıdaki parametrelerin `DEFAULT` profildeki değerlerini kontrol etmek çok önemlidir:

| Parametre | Açıklama | Önerilen Değer |
|-----------|----------|----------------|
| `FAILED_LOGIN_ATTEMPTS` | Hesabın kilitlenmesi için izin verilen hatalı giriş sayısı | 3-5 |
| `PASSWORD_LIFE_TIME` | Şifrenin kaç gün geçerli kalacağı | 90 |
| `PASSWORD_REUSE_TIME` | Aynı şifrenin tekrar kullanılması için geçmesi gereken gün | 365 |
| `PASSWORD_VERIFY_FUNCTION` | Şifre karmaşıklığını kontrol eden fonksiyon (Complexity) | ORA12C_ST_VERIFY |
| `PASSWORD_LOCK_TIME` | Hatalı giriş sonrası hesabın kilitli kalacağı gün sayısı | 1 |

## Kullanıcıya Profil Atama

Bir kullanıcıya belirli bir profili atamak veya mevcut profilini değiştirmek için:

```sql
-- Profil oluşturma
CREATE PROFILE app_user_profile LIMIT
  FAILED_LOGIN_ATTEMPTS 3
  PASSWORD_LIFE_TIME 60
  SESSION_PER_USER 2;

-- Kullanıcıya atama
ALTER USER scott PROFILE app_user_profile;
```

<Callout type="warning" title="Önemli Not">
Kaynak limitlerinin (SESSION_PER_USER, CPU_PER_SESSION vb.) aktif olabilmesi için `RESOURCE_LIMIT` sistem parametresinin `TRUE` olması gerekmektedir:
`ALTER SYSTEM SET RESOURCE_LIMIT = TRUE;`
</Callout>

## Kullanım Senaryoları

### 1. Şifre Politikası Denetimi
Hangi profillerin şifre politikasının limitlenmediğini (`UNLIMITED`) bulmak için:

```sql
SELECT profile, resource_name, limit
FROM dba_profiles
WHERE resource_type = 'PASSWORD'
  AND limit = 'UNLIMITED';
```

### 2. Atıl Oturumların Kapatılması
`IDLE_TIME` parametresi ile belirli bir süre işlem yapmayan oturumların otomatik olarak sonlandırılmasını sağlayarak veritabanı kaynaklarını koruyabilirsiniz.

## Sonuç

Doğru yapılandırılmış profiller, veritabanı güvenliğinin ilk savunma hattıdır. `DEFAULT` profilin çok gevşek tutulması güvenlik açıklarına yol açabilir. Bu nedenle, farklı kullanıcı tipleri (Uygulama, Admin, Raporlama) için ayrı profiller oluşturulması en iyi uygulamadır (Best Practice).
